# Security

Mirante4D is pre-alpha research software. Only the current `main` branch is
maintained; no released version currently receives a separate support window.

Please do not open a public issue for a vulnerability or accidental disclosure.
Use the repository's **Security → Report a vulnerability** route. If that
private route is unexpectedly unavailable, open a public issue containing no
sensitive details and ask the maintainers to restore it.

Include:

- the affected revision;
- reproduction steps;
- likely impact, especially to source data or private metadata;
- any known mitigation.

Reports are handled on a best-effort basis without a guaranteed response SLA.
Ordinary bugs and non-sensitive crashes should use the public issue tracker.
Never attach private microscopy data, credentials, access URLs, or unredacted
machine paths to either route.
