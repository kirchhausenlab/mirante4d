## WP-05 — Documentation And Governance Reset

Goal: replace the fragmented documentation authority chain before new
architecture contracts accumulate.

Required work:

- install the exact concise documentation tree named by the WP-05 entry brief;
- reduce both `AGENTS.md` files to durable enforceable rules and a short read
  order;
- establish one product charter, one factual current-state authority, one
  current-work ledger, one unresolved backlog, and accepted ADRs;
- merge overlapping architecture, concurrency, modularity, errors,
  observability, testing, release, data-safety, and format policies;
- keep target documents clearly distinct from implemented current state;
- remove superseded plans/execution logs from active navigation; and
- add bounded documentation metadata, link, anchor, authority, and index
  checks.

Exit requires a clean-clone contributor to find the product scope, current
state, active work, architecture, commands, and decisions through the stated
read order, with no conflicting active authority.

## WP-06 — Verification Bootstrap

Goal: install fast, independent safety rails before architectural cutovers.

Required work:

- reconcile the pre-foundation inventory and removal ledger so every retained
  test, fuzz target, benchmark, UI snapshot, package smoke, product scenario,
  and audit command is kept with purpose, moved, rewritten, or deleted with a
  reason;
- close the known viewport-report mismatch, recursive/line-limit primary gate,
  stale external-evidence identity, and duplicated slow topology;
- build pure test-state builders and only the minimum temporary adapters, each
  with a named deletion gate;
- implement a fixture registry with archive limits, pre-write safe validation,
  producer/fact/reader independence, authority promotion, immutable digests,
  and expiry for bootstrap-format vectors;
- make ordinary render targets tiny and injectable;
- create six nonrecursive leaves: policy, lint, unit, contract, UI, and
  doctest;
- expose exactly `PR / policy` and `PR / rust` for pull requests and matching
  non-required `Main / ...` results for protected `main`;
- split GPU, E0-E4, package, format-lifecycle, performance, stress, and
  scientific acceptance from the PR profile;
- add workflow/job/process/test timeouts, deterministic grouping/seeds,
  duration output, zero-retry policy, useful logs, and cache isolation;
- delete recursive gate composition, per-test subprocess allowlists, and
  `--all-targets` correctness runs that accidentally execute benchmarks; and
- calibrate twenty consecutive cache-free public attempts. Functional failures
  and censored timeouts remain in the window; only predeclared infrastructure
  failures may be invalidated before inspecting test results.

The public checks target uncached p95 below ten minutes and a fifteen-minute
ceiling. A miss stops the package; it does not enable caching or increase the
budget. Cache/artifact activation is a separate later decision.

Exit requires independent leaf results, exact requirement ownership, complete
inventory disposition, functioning fixture-lineage enforcement, target checks
installed/read back on protected `main`, and exact-revision proof that the four
known verification blockers were replaced rather than renamed or hidden.

WP-06 establishes machinery. It cannot claim target-format T1 conformance
before WP-10A creates the target schema and independent corpus.
